Intro

As we have succefully completed printing a "thing" to the console, that is just written but not stored anywhere, in this tutorial, let's see what Variables are.
Yo! Before going to variables, explain what the "thing" is ?
The thing we have printed earlier is a String. A string is a sequence of characters. This is one type of data type. there are other data types like Int, Double, Char, Long, Short, Byte, String... Data types are used to categorize the set of data and further perform respective operations on data. From the above list of data types, data types that deal with Numbers are,

  • Byte - can store 8-bit Number (from -128 to 127)
  • Short - can store 16-bit Number (from -32768 to 32767)
  • Int - can store 32-bit Number (from -2147483648 to 2147483647)
  • Long - can store 64-bit Number (from -9223372036854775807 to 9223372036854775807)
  • Double - can store 64bit Number (stores double-precision point value)

Type Inferance

As we have discussed in the Advantages Of Kotlin section in the Kotlin Introduction page, Kotlin supports Type Inferance, which means that the variable type is not to be provided, while declaring variables. Compiler sees through the type of the data you assign to the variable and treats the variable as the same.
